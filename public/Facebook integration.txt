Step 1: Access Meta Events Manager

Step 2: Get or Create a Pixel

If no Pixel exists:

Click Connect Data Sources (green button)

Step 3: Generate Conversion API Access Token

In Events Manager, select your Pixel
Go to Settings tab
Scroll down to Conversions API section
Click Generate Access Token
Copy the token (long string starting with EAA...)

Important: This token won't be shown again. Save it somewhere safe before closing.

Step 4: Add Environment Variables
Open apps/dj-yarin/.env.local and add:
# Facebook Pixel (client-side)
NEXT_PUBLIC_FB_PIXEL_ID=123456789012345

# Facebook Conversion API (server-side)
FB_PIXEL_ID=123456789012345
FB_CAPI_ACCESS_TOKEN=EAAxxxxxxxxxxxxxxxxxxxxxxx
Replace with your actual values. Both Pixel IDs are the same number.

Step 5: Restart Dev Server
bash# Stop the current server (Ctrl+C), then:

Troubleshooting
Pixel Helper shows no pixel:

Check NEXT_PUBLIC_FB_PIXEL_ID is set (needs the NEXT_PUBLIC_ prefix)
Restart dev server after adding env vars

Lead event not appearing:

Check Events Manager Рєњ Test Events tab (not Overview)
Verify FB_CAPI_ACCESS_TOKEN is correct
Check server console for error logs

Duplicate leads showing:

Both events should have the same event_id Рђћ Facebook deduplicates automatically
If duplicates appear, check that eventId is passed to both client and server calls

WhatsApp not opening:

Check siteConfig.contact.whatsapp has the correct number format (972529533448)
